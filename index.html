PROJECTS

I've implemented a brute-force ray tracer based on a GitHub repository I found. The code is relatively simple as it doesn't utilize any of the newer features of C++. However, since this project served as a means of learning and understanding the fundamentals of computer ray tracing, much of the code is either challenging or nearly impossible to read. The reason to the aformentioned is that, I deviated from the methodologies outlined in the accompanying book, resulting in missed optimizations and making the code somewhat inefficient.

Ray-Tracing guide/book that was used: https://raytracing.github.io/books/RayTracing

My implimenation: https://github.com/SHamkhalil/Ray-Tracer

I've developed a simple web scraper in Python using the Beautiful Soup library to extract information from a book website (https://books.toscrape.com/), providing me with a significant amount of raw data for analysis. This project was a 'two birds with one stone' type of endeavor, where not only did I gain valuable insights into the Python Matplotlib library, but I also acquired hands-on experience with web scraping algorithms. After acquiring the data, I proceeded to process and visualize it using Matplotlib. The visualization focused on three primary categories: book ratings, prices, and titles, which is reflected in the graph to the right of this text.
